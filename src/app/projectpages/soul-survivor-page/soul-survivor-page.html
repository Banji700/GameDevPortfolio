Soul Survivor

For my project I am going to be making a third person shooter style game that allows you to switch between a sword and a gun. My game is going to be a futuristic themed game set in in the distant future. I decided to make my game a pc game using keyboard and mouse.

The main software I am going to be using to make my game is Unity and using C# for coding although I am going to be primarily using Unity but I will be practicing to use unreal engine as well as I feel like it can help me in the future.

I am going to be using Unity because I am experienced with unity but also hoping to learn new things as I try to use new mechanics in my game.

The main aspect I wanted to focus on was the coding because I wanted to improve my coding skills when I was creating my game. This is going to be the first time I am going to be working with animation and animation states in unity and so I hope I am able to make the animations look smooth and fluid.

Story

The main character P-30 was an experiment to create an evolved human using the advance technology of the future. These experiments were successful, but they left his mind broken and scared. He now believes the humanity needs to be wiped out to create a new peaceful society, but other people don’t see it that way and try to stop him. In his eyes what he is doing is right but in the eyes of others they don’t see it the same way.

Game Process 1 Camera

First thing I did when starting my game was to import my character and add a third person camera and this was tricky for me because it was the first time I used unity's cinemachine camera feature. Cinemachine is basically a set of camera options that allow to have different type of camera settings.
For my camera settings I made it a third person camera. I added a reticle to the middle of that screen you know where you are facing but also the reticle changes when you aim. I also made it so that the camera zooms in when aiming for a more accurate shot.

Game Process 7 Enemy Movement

For my enemies I knew I wanted to them to have a AI system. Unity has a nav mesh feature that allows the enemies to move by themselves but I adjusted it with my code. I used the nav mesh surface so that it shows the places where the enemy can move and then I used the nav mesh agent so the enemy knows where it can go.
