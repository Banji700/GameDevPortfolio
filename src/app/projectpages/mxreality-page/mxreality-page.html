My Time At MXreality

During my time as a unity developer at mxreality my main priority was to develop virtual scenarios. These scenarios would vary so in one scenario you could play as a doctor checking up on a patient or in another scenario you could play as a waiter in training and it would take you through all the steps you would need to do to complete that task




Whiteboxing Process

A storyboard would be given to me and it would be an excel spreadsheet would be with the information that would be expected to be in the scenario. But then from there I would start creating the scenario. If assets and audio for the scenario were made already then I would start using them but if not, I would start white boxing.





Setting Up TaskList

The first type of things I would focus on doing would be setting up a tasklist of tasks that need to be completed in the scenario. After I set up the tasklist i would start setting up functions in the script so things like camera zooms, audio, questions and messages would all be set up in the script.




Interactive Objects

Then I would make sure all the interactive objects could be clicked on by the player. From there I would set up all the panels for the questions, messages and tooltips that would be interacted with by the player.




Finishing Process

After that I would move on to audio and subtitles. Now the main piece of audio would be the dialogue and depending on the scenario would determine how much dialogue there was. After I set up the dialogue I would then set up the subtitles and make sure the dialogue matched the audio. Then I would set up any background audio that need to be in the scenario

After I would set up all those steps and functions the scenarios would be close to completion. Then I would test the scenario rigorously. I would move around the scene to make sure all the colliders were in the right place so you couldn't walk through things like walls tables and other solid objects. I would then make sure all the text were spelled correctly and then double check animations, audio and functions like camera zooms when characters were talking. I would play the scenario and go through it alongside the excel storyboard. Then I would make a build and test it in the web browser and this part was important because sometimes when the scenario was played it would be different in the web browser than it was in unity but after testing it in the web browser it was also important to test it on other laptops,computers and other devices it would be played on because sometimes it would play different. The scenario would also be play tested by another team member just to see if I left out anything during testing or just to have a different point of view on the scenario but after that the scenario would be done.




VR Conversion

After the scenario was made, tested and approved I would then convert it into a VR version. The VR version would be abled to be played in VR headsets. I would have to change somethings because the VR version allowed for the player to be able to move more freely than the desktop version so certain things would have to be changed but after the changes were made a build would be made and tested again then finished and ready to be played by others.
